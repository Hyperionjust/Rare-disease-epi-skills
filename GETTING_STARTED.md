# 安装与上手指南（新手版 · 看这一篇就够）

> 这篇写给**第一次用、或不太懂技术**的朋友。你不需要懂"代码""接口""数据库"——照下面一步步点就行。
> 读完约 5 分钟，跑通一次约 1 分钟。

这个工具叫 **rare-disease-epi（罕见病流行病学情报）**。你对它说一句「**查一下 XX 病的患病率**」，
它就帮你把这个罕见病的**流行病学数据**（有多少人得、患病率/发病率/死亡率、遗传病的携带频率）整理成一份报告，
覆盖 **国际（Orphanet/PubMed）+ 美国 + 中国**，并优先查**国内外诊疗指南/共识**当口径锚点。

它最大的特点是**诚实**：每个数字都标了"这条有多可信"，还把"年份、样本量、查的是哪类患者"摊开给你看；
同一个病在同一个国家查到好几个互相打架的数字时，它**排成对比表解释为什么不同，绝不擅自取个平均糊弄你**；
最后专门给你一张 **「🔴 需人工核对清单」**，把最容易出错、最好你自己再核的地方挑出来。

> ⚠️ 一句话免责：这是**研究/情报参考，不是医疗建议、不是诊断**。报告里凡是标 🔴 的，请务必自己再核一遍再用。

---

## 第 1 步：打开"联网搜索"，并把网站"放行"（最关键，先做这个）

这个工具要去**真实的官方/学术网站**上查（Orphanet、美国 NIH PubMed、ClinicalTrials.gov、中国卫健委等）。
两个开关要先确认：

### 1A. 打开"联网搜索"
进 **设置（Settings）→ 功能 / Capabilities**（不同版本可能叫"工具""Connectors"），把 **联网搜索 / Web search** 打开。
公司统一管理的账号可能要找管理员开。

### 1B. 把它要访问的网站"放行"（最容易卡住，重点看）

很多环境只允许访问"白名单"里的网站。**网站没放行 → 工具查不到、报告一片空白。** 两个选择**任选其一**：

**选择 A（最省事，推荐新手）：把允许访问的网站设成"全部"。** 在网络/域名设置里填一个星号：

```
*
```

一个 `*` 代表"所有网站都允许"，保存即可。

**选择 B（不想全开，就只放行下面这些）：**

```
# —— 工具自动抓数据要用的（必须放行，否则脚本会失败）——
eutils.ncbi.nlm.nih.gov         # 美国 NIH PubMed E-utilities（流行病学文献、指南、GeneReviews）
www.clinicaltrials.gov          # 美国临床试验登记库（入组数作活动量参考）
                                #   ⚠️ 注意一定是带 www. 的；裸域名 clinicaltrials.gov 常被白名单挡掉

# —— Orphanet（罕见病 epi 专用库；很多环境默认没放行，请务必补上）——
www.orpha.net                   # Orphanet 疾病页（患病率、亚型、ICD/OMIM/GARD、指南链接）
www.orphadata.com               # Orphadata（Orphanet 的可下载数据集 / 科学 API）

# —— 补充源 ——
rarediseases.info.nih.gov       # 美国 GARD（罕见病信息中心，美国侧补充）

# —— 中国官方（查中国指南 / 目录病种）——
www.nhc.gov.cn                  # 国家卫健委（《罕见病诊疗指南》2019 + 《86个罕见病病种诊疗指南》2025）
```

> 💡 **Orphanet 特别提醒**：`www.orpha.net` / `www.orphadata.com` 在不少环境里**默认是没放行的**。
> 如果报告里 Orphanet 那部分显示"被拦截/403"，多半就是这个原因——把这两个域名加进白名单再重试即可。
> 实在不想逐个加？直接用**选择 A（填 `*`）**，一劳永逸。
>
> 还有个兜底：就算脚本抓不到 Orphanet，工具也会**自动改用联网抓取**直接读 Orphanet 疾病页
> （`https://www.orpha.net/en/disease/detail/<编号>`），所以即便域名没放行，Orphanet 数据通常也还能拿到。

中国的知网、万方、维普、微信公众号、患者组织白皮书**没有开放接口、工具不会去抓**——这是设计如此（硬抓必然失败、还会误导）。
中国侧工具改为**给你最优检索式**，你把查到的原文/截图发给它，它帮你整理进对比表，并把每条都标红提醒你核对。

## 第 2 步：发起一次查询（就打一句话）

放行好网站后，回到聊天框，**直接用大白话打一句**，比如：

> 查一下脊髓性肌萎缩症（SMA）的患病率
>
> 帮我做一下戈谢病的流行病学数据，分亚型
>
> Pompe 病在中国有多少人？发病率和携带频率帮我看看

病名可以是**中文名、英文名、缩写**，它都能认。
想要一份**能存档、能发同事的好看文档**？加一句：「**给我出一份 HTML / PDF 的可视化报告**」。

## 第 3 步：它会先问你一两个问题（别慌，这是正常的）

它不会马上闷头写一大篇，而是先确认方向：

1. **"你要查的是不是这个病？"** —— 列出病名、别名、编号让你确认，别认错病（有些缩写对到好几个病）。
2. **"主要看患病率、发病率、死亡率，还是携带频率？看国际、美国还是中国？要不要分亚型？"**
   它还会顺带告诉你**找到/没找到官方诊疗指南**（找到的话当作本次口径锚点）。
3. **"这个病的亚型治疗差别大，要不要分亚型分别给数？"**（如有显著分型时）
4. **"要不要一次写完？"** —— 边写边停着问你，还是一口气写完、把要核对的地方红色集中标在最后。不确定就选后者。

你简单回一句，它就开始干活。

## 第 4 步：看懂报告里的"小徽章"（一眼判断哪条能信）

每条信息后面有个小标记，告诉你**这条有多可信**。七个：

| 标记 | 一句话意思 |
|------|-----------|
| 🏛️ 官方 | **最可信**：各国官方疾病登记/流调的**登记数**（如日本受給者証、韩国 KDCA、官方流调）。|
| ✅ 库 | 也很可信：Orphanet、PubMed 等权威**数据库**能查到。|
| 📘 指南 | 诊疗指南 / 专家共识 / GeneReviews（**高等级**）。但它给的数字是二手值，工具会帮你**溯回原始研究**。|
| 🔍 web | 单篇研究 / 联网查到的，**会附 PMID 或链接**。|
| ⚠️ | 来源弱（单中心、小样本、会议摘要），**建议再确认**。|
| 🔴 | **最需要你自己再核**的（中国侧、死亡率、外推估算的数字）。绝不会没来源就瞎给。|
| ❓ N/A | 认真查过、确实没找到（会告诉你查了哪些地方）。|

> 几个重要细节：
> - **中国侧的每一个数字都会标 🔴**——就算来自官方白皮书也一样，因为中国侧样本和口径常不透明。
> - **死亡率**和**由携带频率/发病率推算出来的患病率**也强制 🔴（罕见病死亡登记普遍不全；外推不是实测）。
> - **携带频率 ≠ 患病率**：很多罕见遗传病只有"携带频率"没有"临床患病率"，工具会分开列、绝不混算。

报告**最后那张「🔴 需人工核对清单」是重点**：把全篇红色/问号的地方汇总成一张表，逐条告诉你"为什么要核、去哪核"。

## 第 5 步：一个完整的样例（照着跑一遍）

**单独跑数据脚本**（可选；正常用你不用碰它，直接对 Claude 说话即可）：

```bash
# 以脊髓性肌萎缩症（SMA）为例，Orphanet 编号 83330
python scripts/fetch_epi_data.py "Spinal muscular atrophy" --orphacode 83330 --max-studies 150
```

你会看到一份合并 JSON，里面：

- `deterministic_sources.orphanet`：Orphanet 的患病率带、亚型提示、ICD/OMIM/GARD、指南链接（域名放行时）。
- `deterministic_sources.pubmed`：患病率/发病率/流行病学/死亡率各自的命中数 + 种子 PMID；美国/中国 affiliation 命中；指南、GeneReviews 命中。
- `deterministic_sources.clinicaltrials_gov`：入组数（**仅活动量参考，不是患病率**）。
- `china_manual.*`：知网/万方/公众号/白皮书/卫健委指南的**推荐检索式**（工具不抓，给你去查）。
- `source_status`：每个源成功/失败的原因（失败 = 该源没查到或域名没放行，**绝不等于"病不存在"**）。

**出可视化报告**（Claude 会自动填好 JSON 再调用；你也可手动）：

```bash
python scripts/build_report.py <data.json> -o report.html          # 自包含 HTML，可"打印→存为 PDF"
python scripts/build_report_docx.py <data.json> -o report.docx     # 可编辑 Word（需 pip install python-docx）
```

## 第 6 步：把这个工具装到你的 CLI / 客户端（四种 harness）

本 skill 是**可移植**的：目录形式（`SKILL.md` + `scripts/` + `templates/`），frontmatter 只用最通用的
`name` + `description`（各家都认），脚本纯标准库、纯命令行，不依赖任何一家的内置工具。
把整个 `rare-disease-epi/` 文件夹放到对应位置即可：

| Harness | 安装位置 |
|---------|---------|
| **Claude Code** | 全局：`~/.claude/skills/rare-disease-epi/`（Windows：`%USERPROFILE%\.claude\skills\rare-disease-epi\`）；或某项目：`<项目>/.claude/skills/rare-disease-epi/` |
| **Claude 桌面/网页版** | 设置 / Customize → Skills → Upload skill，上传打包的 `rare-disease-epi.zip`（若只认 `.skill` 后缀，改名即可，内容一样） |
| **Qwen Code** | `~/.qwen/skills/rare-disease-epi/` |
| **Kimi Code CLI** | `~/.kimi-code/skills/rare-disease-epi/` |
| **MiniMax Mini-Agent** | 其 skills 目录下 `…/skills/rare-disease-epi/`（参见该工具文档的 skills 路径） |

> 各家可能有**可选增强字段**（如 Kimi 的 `whenToUse`、Qwen 的 `paths`）。本 skill **不把它们当运行必需**——
> 只靠 `name` + `description` 就能在四家触发与运行。需要的话你可以**额外**加这些字段做增强，但删掉也照样跑。
>
> 装好后，确认 Python 3.8+ 可用（`python --version`）、第 1 步的联网与域名已放行，回聊天框打「查一下 XX 病的患病率」即可。

## 第 7 步：万一"搜不到 / 抓取失败 / 报告一片空白"

八成是第 1 步的网站没放行。照下面修：

1. 回 **设置 → 功能 → 网络访问**，把"允许的域名"改成 `*`（最省事）；或对照 **1B** 补齐，**尤其 `www.orpha.net`、`www.orphadata.com`、`eutils.ncbi.nlm.nih.gov`、`www.clinicaltrials.gov`（带 www）**。
2. 确认**联网搜索**开关是开着的。
3. 保存，回聊天框把「查一下 XX 病」**再发一次**。

> 公司/团队账号的白名单可能被管理员锁住——把 1B 的清单发给 IT/管理员放行即可。
> 记住一条：**某个网站没查到 ≠ 这个病不存在、≠ 没有 epi**。早期/超罕见病在某些库查不到、Orphanet 被域名挡掉，都是正常的环境问题。

## 最后再叮嘱一句

这个工具帮你**快速把一个罕见病的流行病学全貌摸清楚、并诚实告诉你每个数字的口径与哪些别全信**。
它**不替你做医疗或科研结论**。报告里所有 🔴 的地方（尤其中国侧、死亡率、外推值），请你（或专业同事）务必溯到原始研究再用。
