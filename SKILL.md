# rare-disease-epi — 罕见病流行病学情报（International + 美国 + 中国）

这是一套**运行时流程**。当有人说「查一下 X 病的患病率/发病率」时，你（Claude）按本文件从头到尾执行。
本工具看 **国际（Orphanet/PubMed）+ 美国 + 中国** 三个口径，重点补强**中国侧**与**口径不一致**两大痛点。

> ⚠️ 免责：本工具产出**仅供研究/情报参考，非医疗建议、非诊断、非投资建议**。所有 epi 数字必须带徽标、年份、样本量并经人工复核。

> **本工具的价值一句话**：不是给一个患病率数字，而是**把每个数字的口径（年份/样本/患者类型）与可信度摊开，
> 把不确定的清清楚楚交到人手上核对**。宁可 `❓N/A` 也不编。罕见病 epi 本就稀少、口径混乱——诚实标注比凑一个数更有用。

## 1. 最高优先级规则：全局来源标注（badges，七档）

**报告里每一个 epi 数字、每一个结论都必须带来源徽标。** 「所有高幻觉区」和「判定为找不到的区域」一律**显式标注**，
绝不静默省略、绝不无来源直接给数字。

| 徽标 | 含义 | 用法 |
|------|------|------|
| 🏛️官方 | **官方登记·最高可信（仅客观登记数）** | 来源是各国官方疾病登记/监管：日本「指定難病」受給者証所持者数、韩国 KDCA 희귀질환 산정특례统计、各国 CDC/卫生部官方流调、NMPA/CDE 等。**只对客观登记/官方统计数生效**。必须附 URL |
| ✅`<库>` | 权威数据库可核实 | 例：✅Orphanet、✅PubMed(PMID)、✅ClinicalTrials.gov。机器可查到的 curated 估计或检索命中。与 🏛️官方 同属**最高一档** |
| 📘指南/共识 | **诊疗指南/专家共识·高等级** | 国内外临床诊疗指南、专家共识、GeneReviews。与 ✅库/🏛️官方 **同档**。国家卫健委官方指南可**同时叠 🏛️官方**。⚠️ 指南给的 epi 数字是**二手/共识值**，须溯原始研究（见第 3 节） |
| 🔍web | 联网检索到的同行评议研究/二手 | **必须附 URL / PMID**。单篇研究属此档；权威医药媒体亦属此档 |
| ⚠️ | 来源存疑 / 单中心小样本 / 单一弱源 | 建议人工确认；单中心、小 N、会议摘要、二手聚合站 |
| 🔴 | 高幻觉区 或 必经人工确认 | **绝不允许无来源直接给数字**。即使查到也要标，并说明不确定性来源 |
| ❓N/A — 未找到 | 检索过但确实没找到 | **必须写明查过哪些源**（Orphanet/PubMed/指南/中国哪些站）。这本身也要显式标出 |

**可信度排序（事实型字段，由高到低）：** `🏛️官方 ＝ ✅库 ＝ 📘指南 ＞ 🔍web ＞ ⚠️ ＞ ❓N/A`。
查不到就写 `❓N/A` 并注明查过哪些源。**绝不编造数字、年份、样本量或来源。**

### 1.1 一个数字可以**同时**带"来源徽标"和"🔴"
`🏛️/✅/📘` 回答"信源多硬"，`🔴` 回答"这数字本身要不要人工兜底"——两者不冲突。遇到下面 1.2 的强制 🔴 字段就**两个都标**。

### 1.2 强制 🔴 的字段（不管查没查到、不管来源多权威，都标 🔴 并解释不确定性来源）

1. **中国侧的任何 epi 数字/结论** —— 无干净开放 API，天然低可信。**即使拿到官方白皮书，🔴 也不解除**（中国侧样本与口径常不透明）。
2. **任何 mortality（死亡率/死亡登记）数字** —— 罕见病死亡登记普遍不全、漏报严重，跨国可比性差。
3. **任何外推/估算/建模得到的患病率** —— 如"由携带频率推算的理论患病率""由发病率×生存期估算的患病率""全球外推到某国"。
4. **跨研究"挑代表值"的判断** —— 同病多个数字里选哪个当"代表"，本质是 AI 主观取舍。
5. **指南/共识里的 epi 数字若溯不到原始研究** —— 数字可标 📘 但口径不明（见第 3 节）。

> 例：`患病率 1/10,000 🏛️官方(日本受給者証2023) 🔴（死亡相关口径/外推时）`；
> `携带频率 1/42 🔍web(PMID…) ⚠️（单一族群样本）`；中国侧 `发病率 1/9,788活产 🔴 📘(中国SMA共识引) 🔍web(原始研究PMID…)`。

## 2. 暂停门 (Pause Gates) — 必经人工确认后再继续

> 「暂停门」是内部术语，**给用户看到的话里绝不出现这个词**（见 2.1 的重写话术）。

为防止错误向下游传播，**G0–G3 默认硬停**：到达时**停下来、把该门的中间结论给用户、等确认后再继续**。
**唯一例外**：用户开场明确说「全部一次跑完别停」时，G2/G3 降级为"高亮 🔴 + 写进核对清单"（见 2.2），G0/G1 仍至少复述确认一次。

| 门（内部编号） | 位置 | 内部触发逻辑（不直接说给用户） |
|----|------|------|
| 🚧 **G0 疾病身份门** | 开场（疾病消歧） | **硬停**：实体错 = 整篇全错。病名/别名/缩写可能对到多个实体（如"SMA"亦指 Spinal Muscular Atrophy 之外的同缩写病）→ 用 ORPHAcode + OMIM + ICD 消歧，多候选必须让用户选定再跑 |
| 🚧 **G1 范围门** | 开场（口径与范围） | **硬停**：决定全文结构。一次性问清：看哪些 metric（患病率/发病率/死亡率/携带频率）、哪些国家/口径、要不要分亚型。**并顺带说明已找到/未找到官方指南**（找到则作为本次 epi 的共识锚点） |
| 🚧 **G2 亚型门** | 检测到分型差异显著时 | **硬停**：若该病存在**指南分类下治疗方案显著不同的亚型**（尤其单基因病不同基因/突变亚型、或临床分型），停下确认**按哪些亚型拆**（依据=指南认可的分类）。携带频率 vs 临床患病率必须分开 |
| 🚧 **G3 中国门** | 中国侧任何 epi 结论 | **硬停**：无干净源。中国侧每一条都默认等用户到官方站/白皮书核对后再定稿（🔴 不因官方白皮书而解除） |

### 2.1 各门"给用户看到的那句话"——必须用大白话（照下面说，可更自然）

**铁律**：内部逻辑、门编号、流程结构**全部不变**；只把"对用户说出口的话"换成正常人说话。
**禁止**对用户说出这些词：暂停门 / 口径门 / 消歧 / ORPHAcode / metric / proxy。必须用到时就地翻译成人话。

- **G0 身份确认**（替代"疾病消歧"）：
  > 「我先跟你确认一下：你要查的是不是 **脊髓性肌萎缩症（SMA，Spinal Muscular Atrophy，Orphanet 编号 ORPHA:83330 一系，OMIM 253300，ICD-10 G12.0）** 这个病？是的话我就开始查；如果你指的是别的（有些缩写会对到好几个病），告诉我一声。」
  > （若有多个候选：「你给的名字能对到下面几个病，麻烦选一个：1)… 2)… 3)…」）

- **G1 范围**（替代"口径门"）：
  > 「查之前定个方向：① 你主要想看**患病率、发病率、死亡率，还是携带频率**（罕见遗传病常只有携带频率、没有临床患病率，我会注明）？② 重点看**哪些地方**——国际整体、美国、还是中国？③ 这个病有几个**亚型，治疗差别挺大**，要不要我分亚型分别给数？
  > 另外告诉你一声：我**找到了/没找到**这个病的官方诊疗指南（找到的话我会拿它当本次的口径锚点）。」

- **G2 亚型**（替代"亚型门"）：
  > 「这个病按指南分成几个亚型（比如 SMA 的 1/2/3/4 型，按起病年龄和 SMN2 拷贝数分，治疗和预后差别很大）。流行病学最好**分亚型分别给**，不然一个笼统数字会误导。我按指南的分型来拆，你看可以吗？」

- **G3 中国侧**（替代"中国门"）：
  > 「**中国这边的患病率/发病率**——中国没有像 Orphanet、PubMed 那样干净的公开数据库，数据散在知网、公众号、患者组织白皮书里，没有开放接口。所以我会给你**最优检索式**让你（或我）去查，你把查到的原文/截图发我，我帮你整理进对比表；但中国相关的每一条我都会**标红**，建议最后到原始出处再核一遍——就算是官方白皮书也一样标红，因为样本和口径常不透明。」

> 多个门可在同一处一并提请确认（G0+G1+（必要时）G2 开场一起问，用 `AskUserQuestion` 最干净），但未得确认不得跨过该门继续。

### 2.2 "一次跑完"模式 —— 也用大白话问

到场默认走逐门确认。但要**主动用大白话**把"要不要一次跑完"交给用户：

> 「还有：报告里有几个地方特别容易出错、最好你来拍板（亚型怎么拆、中国侧数据）。你想要我**每遇到一个就停下来问你**，还是**一口气先把完整初稿写出来、把所有要你核对的地方都用红色集中标在最后**？（不确定就选后者，省事，最后照红色清单核就行。）」

- 选"逐个停" → 默认逐门硬停。
- 选"一次出" / 用户开场就说「别停直接出」→ 把 **G2/G3 降级为"高亮 🔴 + 写进核对清单"**（G0/G1 仍复述确认一次），
  并在报告顶部声明："⚠️ 已按你的要求一次跑完、没有逐项停下确认，下面所有标红的地方都还没经过你核对。"

## 3. 指南/共识：最高等级来源，但用法有讲究（徽标 📘）

确认疾病身份(G0)后、与主 epi 抓取**并行**，**优先搜索国内外临床诊疗指南/专家共识**（类比肿瘤领域的 NCCN/CSCO）。
它们是高等级来源（📘，与 ✅库/🏛️官方同档），但**用法别用过头**：

### 3.1 国际（多数可自动化，脚本已覆盖）
- **GeneReviews（NCBI Bookshelf）**：专家撰写、按病种、含 Epidemiology 段——遗传罕见病金标准。脚本经 PubMed `"GeneReviews"[Book]` 命中 PMID（如 SMA = PMID 20301526 / NBK1352）。
- **PubMed 指南过滤**：脚本已加发表类型 `Guideline[ptyp]` / `"Practice Guideline"[ptyp]` / `"Consensus Development Conference"[ptyp]`。
- **Orphanet 关联的临床实践指南链接**：随 Orphanet 抓取一并取（脚本 `orphanet.guideline_links`，含 PNDS/HAS、ERN、Orphananesthesia 等）。
- 国际病种基金会 / ERN / NICE 专病共识（PubMed 或 web 补）。

### 3.2 中国（官方文件位置已知 + 学会共识走副驾）
- **国家卫健委官方诊疗指南（最高等级）**：①《罕见病诊疗指南》(2019，覆盖**第一批目录 121 病种**)；②《86 个罕见病病种诊疗指南（2025 年版）》(覆盖**第二批目录 86 病种**)。两批共 **207 种**，北京协和 + 中国罕见病联盟牵头，PDF 在 nhc.gov.cn。
  ⚠️ **仅覆盖这 207 个目录内病种**；目录外的病无官方指南，转学会共识。（脚本 `china_manual.nhc_official_guidelines` 给检索定位。）
- **学会/专病共识**：中华医学会各专科及专病组织的《中国___诊治指南/专家共识》（中华系列杂志/CNKI，部分进 PubMed）。
- **指南聚合库**：医脉通(medlive)、梅斯(medsci.cn/guideline) → 副驾：生成检索式，用户粘贴原文。
- 地方名录如《上海市主要罕见病名录》作补充参考。
> nhc.gov.cn 等政府站可能需加入域名白名单；取不到就给出已知 PDF 链接让用户手动下载。

### 3.3 用法规则（关键，别用过头）
- 📘 是**高等级**，排序与 ✅库/🏛️官方同档；国家卫健委官方指南可**同时叠 🏛️官方**。
- 但指南给的 epi 数字是**二手/共识值**，常不带年份/样本量/患者类型，因此：
  - 指南里的 prevalence/incidence/mortality 数字 → 标 📘，并**顺着其引用溯到原始研究**，用原始研究填对比表的「年份 / 样本量 / 患者类型」三列；
  - 溯不到原始出处 → 这三列写 `❓N/A（指南引用但未溯及原始研究）`，数字仍可标 📘 但注明口径不明。
- 指南对这两件事**最高权威、可直接采信**：① **病例定义/诊断标准**（填对比表第 8 列）；② **指南认可的亚型分类**（直接喂给 G2 亚型门——这就是"按哪些亚型拆"的依据）。

## 4. 两半不对称（本 skill 成败关键）

罕见病 epi 的数据源天然分两半，**绝不能用同一种方式对待**：

**A. 可自动化半边（有 API / 可稳定抓取）→ skill 当"抓取器"，写进 `scripts/fetch_epi_data.py`：**
Orphanet（curated 点估计/亚型/validation）、PubMed E-utilities（MeSH 限定的 prev/inc/mortality + 国别 affiliation + 指南过滤 + GeneReviews）、ClinicalTrials.gov（入组数作 proxy）、GARD（美国侧补充）。

**B. 不可自动化半边（中国）→ skill 当"副驾"，绝不假装能抓：**
知网/万方/维普（付费墙+反爬+无开放 API）、微信公众号（基本不可编程访问）、患者组织白皮书（散落 PDF）、学会共识（部分仅 CNKI）。
**脚本不抓这些**（脚本里它们一律 `null` + 优化检索式）。中国侧定位 = **① 生成优化检索式 → ② 接收用户粘贴/上传原文或截图 → ③ 归一化进对比 schema**。
中国侧所有结论默认 🔴 + 走 G3 中国门（即便给了官方白皮书，🔴 不解除）。

> 此不对称是成败关键：**让脚本去抓知网/公众号必然失败并误导用户**。中国侧 = query 优化 + 结构化对比 + 标红，不是检索。

## 5. 数据脚本（确定性源，永远先跑它）

```bash
python scripts/fetch_epi_data.py "<疾病名>"
python scripts/fetch_epi_data.py "Spinal muscular atrophy" --orphacode 83330
# 可选： --gard-id 7883   --max-studies 200   --no-studies-list   --compact
```

- 纯 stdlib（urllib），无需 pip。返回合并 JSON 到 stdout。**工作流第一步，绝不绕过**（即使要出可视化报告，也先跑脚本）。
- 覆盖**有干净 API 的确定性源**：

| JSON 路径 | 来源 | 喂给哪步 | 徽标 |
|-----------|------|---------|------|
| `deterministic_sources.orphanet` | Orphanet 疾病页 | 身份卡、Orphanet 患病率带、亚型提示、指南链接 | ✅Orphanet（+ 📘 for 指南链接） |
| `deterministic_sources.pubmed.metric_searches` | PubMed E-utilities | 各 metric 的命中计数 + 种子 PMID（prev/inc/epi/mortality） | ✅PubMed → 逐 PMID 读后转 🔍web(PMID) |
| `deterministic_sources.pubmed.country_searches` | PubMed `China[ad]` / `United States[ad]` | 国别 epi 文献定位 | ✅PubMed |
| `deterministic_sources.pubmed.guidelines / genereviews` | PubMed ptyp / `[Book]` | 第 3 节指南锚点 | 📘 |
| `deterministic_sources.clinicaltrials_gov.enrollment_proxy` | ClinicalTrials.gov v2 | **仅活动量 proxy，非患病率** | ✅ClinicalTrials.gov（标注"proxy 非 epi"） |
| `deterministic_sources.gard` | GARD(NCATS) | 美国侧补充 | 🔍web / ❓N/A |

- **关键约定**：任何源失败/查无 → 该源在 JSON 里是 `null`，`source_status` 写明原因。**`null` 永远不等于"病不存在"或"没有 epi"**。
- `china_manual.*` 与 `web_and_manual.*` 脚本一律返回 `null` + 检索式/官网 URL —— 这些走 web + 人工。
- `identity_rollup` 给 G0 消歧用（ORPHAcode/OMIM/ICD/亚型提示/MeSH 翻译）。

### 5.1 网络与 Orphanet 域名（重要，照此处理）
脚本要访问：`eutils.ncbi.nlm.nih.gov`（PubMed）、`www.clinicaltrials.gov`（**必须 www，apex `clinicaltrials.gov` 常被白名单 403**）、`www.orpha.net`/`www.orphadata.com`（Orphanet）、`rarediseases.info.nih.gov`（GARD）。
- 若 `source_status.orphanet` 报 **blocked/403**（很多环境 Orphanet 域名未放行）：**这是环境设置问题、不是病没有 epi**。两条路任选：
  1. 把 `www.orpha.net` / `www.orphadata.com` 加入域名白名单（见 `GETTING_STARTED.md`）后重跑；**或**
  2. **直接用 `web_fetch` 抓 Orphanet 疾病页**（`https://www.orpha.net/en/disease/detail/<ORPHAcode>`）——该页**服务端渲染**，可直接读到 Prevalence、Epidemiology 段、validation、各亚型 ORPHAcode、指南链接。脚本 `web_and_manual.orphanet_manual` 已写明此回退。
- 若**所有源都 error**：多半是环境没放开联网域名，**不是病不存在**——见 `GETTING_STARTED.md` 的"域名放开"。

### 5.2 开跑前的"网站放行确认"（首次安装 / 每个新会话第一次查时，必做）

>
> 选项：**① 已放行 / 用了 `*`（直接开查）** ｜ **② 不确定，给我清单和怎么设** ｜ **③ 还没弄**

- 选 ① → 直接进 G0 开查。
- 选 ②/③ → 先给出放行清单与一键做法（**别让用户自己翻文档**），然后再开查：
  > 必放行：`eutils.ncbi.nlm.nih.gov`、`www.clinicaltrials.gov`（**带 www**）、`www.orpha.net`、`www.orphadata.com`、`rarediseases.info.nih.gov`；中国指南另加 `www.nhc.gov.cn`。

## 6. 对比 schema（同病同国出现多个数字时——本 skill 的核心产出）

当**同一疾病、同一国家/口径**查到多个数字时，排成对比表，**列的优先顺序严格如下**（前三列是第一屏决策依据）：

| 列 | 字段 | 说明 |
|----|------|------|
| 1 | **年份** | **数据采集年份**；与发表年份不同时**都列**（如"采集 2015–2018 / 发表 2021"）|
| 2 | **样本量 / survey size** | N + 调查覆盖人口 base（如"N=120 例 / 覆盖 2,300 万活产"）|
| 3 | **患者类型 / 纳入范围** | 是否只看特定患者：某院就诊者 vs 全人群；确诊标准；是否限定某亚型/年龄段/线别 |
| 4 | metric 类型 | point/period/lifetime prevalence ｜ incidence（发病率/出生患病率）｜ mortality ｜ **carrier frequency（携带频率）**。**口径不同分开列，绝不混算** |
| 5 | 数值 + 置信区间/范围 | 如"1/10,000 (95%CI 1/8,500–1/11,800)"或 Orphanet 带"1-9/100,000" |
| 6 | 地理层级 | 全国 ｜ 单省/单地区 ｜ 单中心 |
| 7 | 分母人口 | per 100,000 / per million / 活产数（live births）/ 新生儿筛查人数 |
| 8 | 病例定义 / 诊断标准 | 遗传病注明**按基因型还是表型**；**有指南时以指南口径为准（指南最高权威）** |
| 9 | 研究设计 | 登记(registry) / 横断面 / 单中心回顾 / 新生儿筛查 / meta / 建模外推 |
| 10 | 来源 + 徽标 | PMID/URL + 徽标（中国侧叠 🔴；mortality 叠 🔴；外推叠 🔴）|

表后给**冲突解读**（必写）：解释数字为何不同（多为**年份、样本量、患者类型**差异——对应前三列）、哪个口径更可信、适合什么用途。
**绝不擅自把多个数字合并成一个平均数。** 若要给"代表值"，必须显式标 🔴 并说明是 AI 取舍、列出被舍弃的数字。

## 7. 亚型处理（遗传/指南分型）

检测该病是否存在**指南分类下治疗方案显著不同的亚型**（尤其单基因病的不同基因/突变亚型，或临床分型）。若存在（→ G2 亚型门）：
- **分亚型分别给 epi**，不要笼统成一个数字。亚型分类**以指南/Orphanet 为准**。
- **单基因病必须区分 carrier frequency（携带频率）vs 临床 prevalence**——很多罕见病只有携带频率、无临床患病率，**必须注明、绝不混用**（携带频率 ≠ 患病率；由携带频率推算的患病率属外推 → 🔴）。
- Orphanet 中这类常有**独立 ORPHAcode/亚型条目**，按亚型映射（脚本 `identity_rollup.subtype_hint` 会提示该 ORPHAcode 是亚型还是母病）。
- 若仅有全病种合并数字、无法拆分 → 明确写"仅有合并数据、无法按亚型拆分"并标 🔴。
- 输出**亚型矩阵**：行=亚型，列=各国/各口径，格=数值+徽标+携带/临床标注。

## 8. 工作流（每步产出都带徽标）

> 能用脚本 ✅ 的优先 ✅；指南/共识/GeneReviews → 📘；逐篇 PubMed 研究 → 🔍web(PMID)，单中心小样本叠 ⚠️；
> 中国侧 / mortality / 外推走 🔴（**🔴 不因来源是官方/指南而解除**）。

0. **网站放行确认（首次/每会话第一次）**：跑脚本前先按 §5.2 用大白话问一句"所需网站放行了吗"，没放行就先给清单/`*` 做法再开查。
1. **疾病身份确认（🚧 G0）**：脚本 `identity_rollup` + Orphanet 给 ORPHAcode/OMIM/ICD-10/11/GARD/别名/分类层级。多候选让用户选。输出身份卡。
2. **找指南/共识（第 3 节，与 3–6 并行）**：GeneReviews + PubMed 指南 + Orphanet 指南链接 + 中国卫健委/学会共识定位。确定本次**共识锚点**与**病例定义/亚型分类**。G1 时告知用户找到与否。
3. **范围与口径确认（🚧 G1）**：metric / 国家 / 是否分亚型。
4. **亚型检测与拆分（🚧 G2）**：按第 7 节。携带频率 vs 临床患病率分列。
5. **国际/全球 epi**：Orphanet 患病率带（✅Orphanet，注明是 curated 估计、须溯源）+ PubMed prev/inc/epi 种子 PMID（逐篇读出年份/N/患者类型 → 🔍web(PMID)）。
6. **美国 epi**：PubMed `United States[ad]` 命中 + 官方流调/登记（有则 🏛️官方）+ GARD 补充。
7. **中国 epi（🚧 G3 · 副驾）**：脚本给检索式 → 用户粘贴知网/公众号/白皮书原文 → 你归一化进对比 schema。每条 🔴。优先用中国卫健委指南/学会共识里的中国数据（📘），并溯其引用的原始中国研究。
8. **对比表 + 冲突解读（第 6 节）**：同病同国多数字排表，解读为何不同，不取平均。
9. **死亡率（若 G1 要）**：整项 🔴（罕见病死亡登记普遍不全）；给数字必附来源 + 漏报说明。
10. **汇总 → 末尾「🔴 需人工核对清单」（第 10 节，最关键产出）。**

### 9.B 可视化报告（HTML / PDF / Word）

触发可视化后，**先照常跑脚本 + web + 指南 + 各暂停门**拿到全部内容，再用生成器出图。**不牺牲严谨性**。

```bash

python scripts/build_report_docx.py <data.json> -o <output.docx>
```

5. **参考来源（Reference list）**：全文所有 source 链接（PMID/URL）统一收口到最后一个 section，编号 + 可点击 + 徽标。

## 10. 末尾「🔴 需人工核对清单」生成规则（最关键产出 · 两种模式都必出）

报告末尾**必须**生成一节 `🔴 需人工核对清单`，把全篇**所有 🔴 / ⚠️ / ❓N/A 汇总成 checklist**。每条含：

```markdown
| # | 字段 | 当前值/状态 | 为什么要核 | 建议去哪个源核 |
|---|------|------------|-----------|--------------|
| 1 | 中国 患病率 | <数字或❓N/A> | 无开放API、副驾归一化、样本口径不透明 | 知网原文 / 卫健委指南引文 / 患者组织白皮书原始研究 |
| 2 | 任何 mortality | <数字或❓N/A> | 罕见病死亡登记普遍不全、漏报 | 原始死因登记研究 / 国家死因监测 |
| 3 | 外推/建模患病率 | <数字> | 由携带频率/发病率推算，非实测 | 原始建模假设 + 实测研究 |
| 4 | 指南引用但未溯及原始研究的 epi | <数字 📘> | 二手共识值、年份/样本/患者类型不明 | 指南参考文献 → 原始研究 |
| 5 | 跨研究"代表值"取舍 | <选定值> | AI 主观，列出被舍弃数字 | 领域专家 / meta 分析 |
| 6 | 亚型拆分 | <矩阵或"无法拆分"> | 仅合并数据时 | Orphanet 亚型条目 / 指南分型 |
| … | 任何 ❓N/A | 查过未找到 | 显式列出查过的源 | 列出已查的源 |
```

清单要把**所有**带 🔴/⚠️/❓N/A 的字段都收进来：**中国侧每一项、每个 mortality、每个外推值、每个携带频率→患病率换算、每个指南未溯源数字**，不得遗漏。

## 12. 自动化可靠性（执行时心里有数）

- **稳（脚本 ✅，可靠自动化）**：Orphanet 患病率带/亚型/指南链接（域名放行时）、PubMed 各 metric 命中计数与种子 PMID、国别 affiliation 计数、指南/GeneReviews 命中、CT.gov 入组 proxy。
- **高等级但须溯源（📘）**：诊疗指南/专家共识/GeneReviews 的 epi 数字——可信但二手，须溯原始研究填年份/样本/患者类型三列；病例定义与亚型分类可直接采信。
- **每次都靠 web、可能不稳（🔍web/⚠️）**：逐篇 PubMed 研究读取、二手医药媒体。
- **强制 🔴（不因来源权威而解除）**：**整个中国侧**、所有 mortality、所有外推/建模患病率、跨研究代表值取舍、指南未溯源数字。
- 失败处理：脚本任一源 `null` → 当作"该源没查到"，继续其它源与 web；**绝不**因一源失败（或环境域名未放开、或 Orphanet 被 403）就下"病不存在 / 无 epi"。Orphanet 被挡 → 用 `web_fetch` 回退。

> 再次强调本工具的价值：不在"给一个患病率数字"，而在**诚实标注每个数字的口径与可信度**，并把不确定的清清楚楚交到人手上核对。宁可 `❓N/A` 也不编。
